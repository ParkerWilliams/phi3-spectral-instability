# Phase 1 Data Model: Headless Simulation Run with Telemetry

Entities derive from the spec's "Key Entities" and the schemas in
`docs/telemetry.md` (schema_version `1`, the binding contract). Field types use
JSON terms. Validation rules cite the FR/SC they enforce.

---

## Entity: Run

One instance of the agent playing one map start-to-finish. Not serialized as its
own record — it is the *process* that produces a Per-run Summary (1) and an Event
Stream (1). Modeled here because it owns identity, configuration, and lifecycle.

| Field | Type | Notes |
|---|---|---|
| `run_id` | UUID string | Unique per run (FR-009). Generated by the harness. |
| `batch_id` | UUID string \| null | Optional grouping; defaults to a single-run batch (R10). |
| `seed` | integer | Per-run RNG/grouping seed (FR-016); surfaced in `level_start`. |
| `map` | string | BSP filename without extension (FR-011). |
| `config` | AgentConfiguration | The clamped `bot_*` set actually applied. |
| `time_limit_sec` | number | Configurable session cap (FR-003); drives `timeout`. |
| `outcome` | enum | Terminal state (see state machine). |

**Output path** (FR-009): `sims/results/<batch_id>/<run_id>.summary.json` and
`sims/results/<batch_id>/<run_id>.events.jsonl`. Must not collide with another
run; the `sims/results/` tree is gitignored.

### Outcome state machine

```
                ┌───────────────► completed   (level_end{completed} seen, clean exit)
   running ─────┼───────────────► died        (agent death, level ends)
                ├───────────────► timeout      (sim_time_limit reached OR watchdog kill)
                └───────────────► error         (launch/load/crash failure)
```

Rules:
- Exactly one terminal outcome per run (FR-003).
- `completed` requires **both** a terminal `level_end{outcome:"completed"}`
  event **and** a clean engine exit (FR-010, SC-006). Process-exit-0 alone is
  insufficient.
- `error` ⇒ either an `error` summary (if producible) **or** no summary + a
  non-zero process exit and a diagnostic; never a `completed` summary (FR-010).
- Interrupted/killed mid-run ⇒ partial output marked distinct from a clean run
  (edge case); never reported as `completed`.

## Entity: AgentConfiguration

The `bot_*` tunables (catalogue: `docs/bot-stats.md`) applied for a Run, plus the
derived hash. Produced by the harness from supplied inputs.

| Field | Type | Notes |
|---|---|---|
| `bot_config` | object<string, number\|bool> | Each catalogued `bot_*` cvar → its **clamped** value (FR-007). |
| `config_hash` | sha256 hex string | Over canonical (sorted-key) JSON of `bot_config` (FR-007). |

Validation:
- Every value is clamped to its documented range in `docs/bot-stats.md`
  **before** the cvar is set; the clamped value is what appears in `bot_config`
  (FR-008 — no silent out-of-range acceptance; US3 scenario 3).
- Identical `bot_config` ⇒ identical `config_hash`; any difference ⇒ different
  hash (US3 scenario 2).
- Types follow the catalogue: `float` (e.g. `bot_accuracy` 0.0–1.0), `int`
  (e.g. `bot_reaction_ms` 50–1000), `bool` (e.g. `bot_rocket_jump`).

## Entity: Per-run Summary

The aggregated machine-readable result. One JSON file per run. Schema:
`contracts/summary.schema.json` (mirrors `docs/telemetry.md`).

| Field | Type | Source / Rule |
|---|---|---|
| `schema_version` | integer (==1) | FR-012. |
| `run_id` | UUID | FR-004. |
| `batch_id` | UUID \| null | FR-004 / R10. |
| `config_hash` | sha256 | FR-004 / FR-007. |
| `started_at` | ISO-8601 | Wall-clock, harness-owned (FR-004). |
| `ended_at` | ISO-8601 | Wall-clock, harness-owned (FR-004). |
| `duration_sec` | number | `ended_at - started_at`; ≥ 0. |
| `map` | string | BSP stem (FR-011). |
| `outcome` | enum `completed\|died\|timeout\|error` | FR-004 (state machine). |
| `bot_config` | object | The clamped AgentConfiguration (FR-007). |
| `stats` | StatsBlock | Aggregated from the Event Stream (FR-006). |

### StatsBlock (aggregated from events — FR-006)

| Field | Type | Aggregation rule |
|---|---|---|
| `kills` | int | count of `kill` events (== SC-003). |
| `deaths` | int | count of agent `death` events. |
| `damage_dealt` | int | Σ `hit.damage`. |
| `damage_taken` | int | Σ damage to the agent. |
| `secrets_found` | int | count of `secret` events. |
| `secrets_total` | int | secrets present in the map (0 if none). |
| `items_collected` | int | count of `pickup` events. |
| `shots_fired` | int | count of `shot` events. |
| `shots_hit` | int | count of `hit` events. |
| `accuracy` | float, 4 dp | `shots_hit / shots_fired`; **0** when `shots_fired == 0** (zero-denominator edge case → 0, not error). |
| `time_to_exit_sec` | number \| null | sim-time of `level_end` when `completed`; null otherwise. |
| `weapon_usage` | object<weapon, {shots,hits,damage}> | grouped by weapon stem (FR-011). |
| `deaths_by_cause` | object<cause, int> | grouped by `death.cause`. |

Reconciliation (FR-006 / SC-003): counts and rates derived from the events
**must** equal the summary within documented rounding (floats to 4 dp,
`docs/telemetry.md`). Zero discrepancies beyond rounding is the SC-003 bar.

## Entity: Event

A single observable occurrence during a Run. One JSON object per line in the
JSONL stream. Schema: `contracts/event.schema.json`.

| Field | Type | Rule |
|---|---|---|
| `schema_version` | integer (==1) | FR-012 (carried per file; see contract). |
| `t` | number | **seconds since `level_start`** (FR-011), not wall-clock. ≥ 0. |
| `run_id` | UUID | ties event to its Run. |
| `type` | enum | one of the event types below (FR-005). |
| `data` | object | type-specific payload (below). |

### Event types & payloads (`docs/telemetry.md`)

| `type` | `data` payload | Required for |
|---|---|---|
| `level_start` | `{ map, seed }` | first event (US2 sc.1); FR-016 carries `seed`. |
| `level_end` | `{ outcome, time_sec }` | last event (US2 sc.1). |
| `kill` | `{ victim, weapon, distance }` | FR-005; weapon/monster stems (FR-011). |
| `death` | `{ cause, killer }` | `died` outcome (edge case). |
| `shot` | `{ weapon, target\|null }` | FR-005; `accuracy` denominator. |
| `hit` | `{ weapon, target, damage }` | FR-005; `accuracy` numerator. |
| `pickup` | `{ item, value }` | FR-005. |
| `secret` | `{ secret_id }` | FR-005; increments `secrets_found`. |

Ordering & integrity:
- First event is `level_start`, last is `level_end` (US2 sc.1).
- Events are append-only, one per line; the stream must not corrupt/truncate on
  high volume (edge case — log all, no sampling this slice).
- Timestamps monotonic non-decreasing in `t`.

## Relationships

```
Batch (batch_id) 1───* Run (run_id)
Run 1───1 AgentConfiguration  (config_hash)
Run 1───1 Per-run Summary
Run 1───1 Event Stream 1───* Event
Per-run Summary.stats ◄── aggregation ── Event Stream   (FR-006 / SC-003)
```

The Summary's `stats` is a **pure function** of the Event Stream (plus
map-static `secrets_total`); this is what makes reconciliation checkable and the
single source of truth the event stream.
